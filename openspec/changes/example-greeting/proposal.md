# Proposal — example-greeting

> Stage S of QRSPI. Generated 2026-06-18.

## Why

Contributors landing on a fresh clone of the QRSPI kit have no single command
to orient themselves. The `/qrspi:greeting` command closes this gap by emitting
a short context-aware banner (contributor name, branch, kit version, active
change IDs) with graceful degradation when git context is unavailable. Because
the sync generator already handles `claude/commands/*.md` automatically, a
matching Copilot prompt is produced at no extra cost.

## What Changes

- **New:** `claude/commands/greeting.md` -- the `/qrspi:greeting` slash command.
- **New:** `copilot/prompts/qrspi-greeting.prompt.md` -- auto-generated by the
  sync pipeline; no manual authoring required.
- **New:** `openspec/specs/greeting/spec.md` -- canonical spec for the new
  capability.

## Capabilities

### New Capabilities

- `greeting`: A slash command that prints a context-aware welcome banner showing
  the contributor's git user name, current branch, kit version from `plugin.json`,
  and list of active QRSPI change IDs -- creates `specs/greeting/spec.md`.

### Modified Capabilities

- _none_
