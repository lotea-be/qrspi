# Worktree — kit-quality-hardening

> Stage W of QRSPI. Generated 2026-06-18.
> Vertical slices, not horizontal layers.

## Overview

This is a tooling/markdown kit, not a product app. There is no mock-API →
frontend → real-DB progression; instead each slice delivers a thin end-to-end
cut that ends in a demoably green gate: a CI job that goes red-then-green, a
generator run that produces exactly the right tree, or a passing `openspec
validate`. The five slices follow the design's hard-dependency chain: Slice A
(the Node generator + drift gate) is the foundation everything else builds on,
because the drift check is the mechanical contract the other slices rely on to
prove their changes landed. Slice B (opsx removal) comes next because the lint
job in Slice C would otherwise fire on files that are still present. Slices D
and E are independent of each other and can be swapped, but E's validate gate is
a CI job that only makes sense once governance docs and the reference example
exist together. Each slice is one commit on the single PR (matching the design's
sequencing recommendation: one PR, five commits, reviewed slice-by-slice). Slices
A and D have novel judgment-heavy work (Node port, DRY refactor); Slices B, C,
and E are mechanical once A is in place.

## Slices

### Slice A — Node generator + drift gate

The kit's `copilot/` sync tool is ported from `sync-copilot.ps1` / `sync-copilot.sh`
to `sync-copilot.mjs` (a committed Node.js ES module, no build step). The ported
generator carries all robustness requirements that were missing from the PowerShell
script: a source guard that aborts before wiping `copilot/` if any of
`claude/agents/`, `claude/commands/`, `claude/skills/` is missing or empty; a
`--check` mode that exits 1 on any drift (including files deleted from generation,
detected via union-of-trees comparison); try/finally temp-dir cleanup in
`--check` mode; per-line diff output; and a missing-`SKILL.md` warning that
increments an error counter and exits non-zero after finishing. After porting,
`copilot/` is regenerated by running `node sync-copilot.mjs` and the result is
committed. The old `.ps1` and `.sh` files are deleted. The CI workflow file
(`.github/workflows/ci.yml`) is added with just the `drift` job active (the
`lint` and `validate` jobs are added in later slices; they can be stubbed as
`echo "TODO"` here to keep CI green while the workflow file is live). A human
verifies the gate by introducing one-line drift into any `copilot/` file,
running `node sync-copilot.mjs --check`, and confirming it exits 1; then
reverting the drift and confirming it exits 0.

Files touched:
- `sync-copilot.mjs` — new (Node.js port of the full generator)
- `sync-copilot.ps1` — deleted
- `sync-copilot.sh` — deleted
- `copilot/` — fully regenerated (bulk diff against prior PowerShell output is the review surface)
- `.github/workflows/ci.yml` — new (drift job + stubbed lint/validate jobs)
- `CLAUDE.md` — update `sync-copilot.ps1` / `sync-copilot.sh` references to `sync-copilot.mjs`

- M: no mock layer — this is a pure tooling slice with no service/API/UI
- F: n/a
- D: n/a
- T: checkpoint serves as the test: `node sync-copilot.mjs --check` must exit 0 on the committed tree, and exit 1 when drift is introduced manually
- **Model:** opus — first-of-kind Node.js port requiring faithful replication of the PowerShell transform contract (frontmatter rewriting, tool-name mapping, path normalization, agent/instruction file generation); also implements the union-of-trees deleted-file detection, which is non-obvious concurrency logic in the generator's check mode
- Checkpoint: (1) `node sync-copilot.mjs` runs without errors on a clean clone; (2) `node sync-copilot.mjs --check` exits 0 on the committed `copilot/`; (3) introducing a one-line change to any `copilot/` file and re-running `--check` exits 1 with per-line diff output; (4) removing a `claude/commands/` source file and running `--check` exits 1 (deleted-file detection); (5) CI drift job passes on the branch

### Slice C — pin lint + frontmatter/name/heading lints

The CI `lint` job (stubbed in Slice A) is implemented with three checks: (1) a
pin-drift assertion that collects every hand-maintained occurrence of the OpenSpec
version string (excluding `generatedBy:` lines in OpenSpec-generated skill files)
and fails if any occurrence disagrees with the others; (2) a frontmatter/name lint
that verifies every agent has `name:` and `description:` frontmatter, every
command has `description:`, every skill `SKILL.md` has `name:` and `description:`,
every `agent:` reference resolves to an actual agent file, every `model:` field
uses an alias (not a pinned model id), and every `Load skill X` reference resolves
to a real `claude/skills/<X>/SKILL.md`; (3) a heading-level skeleton check that
verifies each canonical section heading from `openspec-templates/*.template.md`
also appears in the corresponding inline skeleton in the relevant agent file. The
README's false "two coupled places" claim and its stale `claude/commands/qrspi:init.md`
path (now `init.md`) are also corrected in this slice, since fixing them is the
immediate payoff of having the pin lint in place.

This slice also absorbs a scope amendment surfaced during Slice A: the
dev-tooling files that drive the sync (`.claude/commands/qrspi-sync-copilot.md`
and `.claude/skills/qrspi-sync-copilot/SKILL.md`) still invoke the deleted
`./sync-copilot.ps1`/`.sh` and are updated to `node sync-copilot.mjs` here,
alongside the README prose fixes (same "stale references to the old script" theme).

Files touched:
- `.github/workflows/ci.yml` — replace stubbed lint job with the real implementation (or reference a reusable script)
- `scripts/lint.mjs` (or similar) — new Node.js lint script implementing the three checks, invoked by CI
- `README.md` — correct "two coupled places" claim, fix stale `claude/commands/qrspi:init.md` path, rewrite pin-bump procedure to use `node sync-copilot.mjs`
- `.claude/commands/qrspi-sync-copilot.md`, `.claude/skills/qrspi-sync-copilot/SKILL.md` — update sync invocation to `node sync-copilot.mjs` (scope amendment from Slice A)

- M: no mock layer
- F: n/a
- D: n/a
- T: checkpoint: introduce a deliberate pin mismatch in one file; CI lint job exits non-zero and names the file; restore the pin and confirm lint passes
- **Model:** sonnet — the lint logic is structured pattern-matching (grep + compare); the heading check is a set-inclusion test; no novel judgment needed once the spec is clear
- Checkpoint: (1) `node scripts/lint.mjs` (or the equivalent) exits 0 on the clean branch; (2) manually changing the OpenSpec pin in one file from `1.4.1` to `1.5.0` causes the lint script to exit 1 and name the mismatched file; (3) adding a `Load skill does-not-exist` line to any agent and running lint exits 1; (4) CI lint job passes on the branch; (5) README no longer says "two coupled places" and the `qrspi:init.md` path is corrected

### Slice D — choreography DRY + tool-grant audit

the next-stage handoff procedure, and the Glob-based precondition pattern (plus
the backlog-atomicity reminder). Each of the eight stage command files
(`questions.md`, `research.md`, `design.md`, `structure.md`, `plan.md`,
`worktree.md`, `implement.md`, `pr.md`) is edited to keep only a thin inline stub
